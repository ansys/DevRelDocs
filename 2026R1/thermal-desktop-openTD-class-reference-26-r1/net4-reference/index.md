# Introduction

This documentation contains the API reference for OpenTD .NET 4 framework, providing a comprehensive overview of all available types and methods. For an introduction to key concepts and a getting started tutorial, please consult the "Getting started" section.

## Namespaces

The OpenTD API is organized into the following namespaces:

- [OpenTD](OpenTD.md) - Core classes, structs, interfaces, and enums for OpenTD functionality
- [OpenTD.AddIn](OpenTD.AddIn.md) - Add-in development utilities
- [OpenTD.Dimension](OpenTD.Dimension.md) - Dimensional analysis and unit handling
- [OpenTD.FloCAD](OpenTD.FloCAD.md) - Fluid modeling components
- [OpenTD.Internal.Communication](OpenTD.Internal.Communication.md) - Internal communication interfaces
- [OpenTD.Logging](OpenTD.Logging.md) - Logging functionality
- [OpenTD.PostProcessing](OpenTD.PostProcessing.md) - Post-processing and data analysis
- [OpenTD.RadCAD](OpenTD.RadCAD.md) - Radiation modeling components
- [OpenTD.RadCAD.FEM](OpenTD.RadCAD.FEM.md) - Finite element mesh components for radiation
- [OpenTD.RadCAD.FEModel](OpenTD.RadCAD.FEModel.md) - Finite element model structures
- [OpenTD.RadCAD.FdSolid](OpenTD.RadCAD.FdSolid.md) - Finite difference solid modeling
- [OpenTD.UserInterface](OpenTD.UserInterface.md) - User interface components
- [OpenTD.UserInterface.Forms](OpenTD.UserInterface.Forms.md) - User interface forms and dialogs
- [OpenTD.UserPreferences](OpenTD.UserPreferences.md) - User preferences and settings
