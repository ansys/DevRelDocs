the log files generated by a process and to display those in the UI. This example demonstrates how to read a log file and display its content in the solution UI.

To display logs while a process is running, work through the sequence of tabs provided.

### 1️⃣ Backend

On the backend, create the solution definition.

### Define the step model

First, create a `StepModel` class named `ProcessLogsStep` with a transaction method to generate a log file.

For the purpose of this example, create a file and write the current time in it.
The `ProcessLogsStep` class contains two fields:

* `wait_time`: a float value that represents the time to wait
* `log_file`: a `FileReference` field to store the log file

In the code snippet, note the following highlighted lines of code:

* `@long_running`: This is a decorator that specifies that the method is asynchronous. This is important because you want to continuously display the content of the log file in the UI while the process is running.
* `self.transaction.upload(["log_file"])`: This uploads the log file from the method directory to the project directory while the process is running.

```python
import time
from ansys.saf.glow.solution import StepModel, StepSpec, transaction, FileReference,long_running

class ProcessLogsStep(StepModel):
    """Process Logs step model."""

    wait_time: float = 10
    log_file: FileReference = FileReference("my_process_logs.txt")

    @transaction(self=StepSpec(upload=["log_file"], download=["wait_time"]))
    @long_running
    def my_process(self) -> None:
        """Wait for some time."""
        start_time = time.time()
        while (time.time() - start_time) < self.wait_time:
            file = open(self.log_file.path, "a")
            current_time = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
            file.write(current_time + "\n")
            file.close()
            self.transaction.upload(["log_file"])
            time.sleep(1)
```

### 2️⃣ Frontend

On the frontend, create a simple button to trigger the `my_process` transaction, read the log file, and display the content in the UI.

### Create the page layout with the process logs

```python
from ansys.saf.glow.client.dashclient import DashClient, callback

from dash_iconify import DashIconify

from ansys.solutions.examples.solution.definition import ExamplesSolution

def layout(step: ProcessLogsStep):
    """Layout of the process logs page."""
    return html.Div(

                className="lead",

            html.Hr(className="my-2"),
            html.Br(),
            dmc.Button(

                id="start_button",
                variant="filled",
                radius="xl",
                style={"color": "#FFFFFF", "width": "20%"},
                leftIcon=DashIconify(icon="streamline:startup-solid"),
            ),
            html.Div(
                id="log_container",
                children=get_logs(step),
                style={
                    "height": "600px",
                    "width": "100%",
                    "overflowY": "scroll",
                    "border": "1px solid #d9d9d9",

```

**Display the log file**

The `html.Div` component with the ID `log_container` displays the content of the log file. The `get_logs` method is responsible for reading the log file and returning its content.

```python
def get_logs(step: ProcessLogsStep) -> html.Div:
"""Return the children containing the lsdyna logs."""
if step.log_file.exists():
    try:
        content = step.log_file.read_text()
    except Exception as e:
        return
    return html.Div(
        [
            html.P(
                "Process logs:",
                className="lead",
                style={"font-size": "20px"},
            ),
            html.Pre(
                content,
                style={"whiteSpace": "pre-wrap", "wordBreak": "break-all", "fontSize": "10px"},
            )
        ]
    )
return html.Div("No logs available yet.")
```

**Define the callback interval**

The `dcc.Interval` component plays a crucial role in this implementation. When activated, it triggers the callback function to update the content of the log file.

To control the activation/deactivation of the interval, create a helper function called `process_in_progress`. Indeed, the interval component is only needed when the process is running.

Within the `layout` function’s `dcc.Interval` component, the `disabled` property invokes the `process_in_progress` function.

Once the transaction completes, the interval is disabled. It is a good practice to disable the interval when it is not needed to avoid unnecessary calls to the backend.

```python
def process_in_progress(step: ProcessLogsStep) -> bool:
"""Return True if the process is in progress."""
return (
    True
    if step.get_long_running_method_state("my_process").status == MethodStatus.Running
    else False
)
```

### Connect the button to the transaction method

When the Start button is clicked, the `my_process` transaction method is  triggered. For this, create a callback function to call the transaction method and activate the interval component.

```python
@callback(
    Output("interval", "disabled"),
    Input("start_button", "n_clicks"),
    State("url", "pathname"),
    prevent_initial_call=True,
)
def start_transaction(n_clicks, pathname):
    """Start the transaction."""
    if ctx.triggered_id == "start_button" and n_clicks:
        project = DashClient[ExamplesSolution].get_project(pathname)
        step = project.steps.process_logs_step
        step.my_process()
        return False
    raise PreventUpdate
```

Finally, create a second callback function to update the content of the log file while the process is running.

```python
@callback(
    Output("log_container", "children"),
    Output("interval", "disabled"),
    Input("interval", "n_intervals"),
    State("url", "pathname"),
    prevent_initial_call=True,
)
def update_logs(n_intervals, pathname):
    if ctx.triggered_id == "interval" and n_intervals:
        project = DashClient[ExamplesSolution].get_project(pathname)
        step = project.steps.process_logs_step
        if process_in_progress(step):
            disable_monitoring = False
        else:
            disable_monitoring = True
        return get_logs(step), disable_monitoring
    raise PreventUpdate
```
