# *class* ans_mat_materials_provider(abstract_field_support: object = None, unit_system_or_result_info: object = None, skip_missing_materials: object = None, Engineering_data_file: object = None, config: OperatorConfig = None)

Provide materials through ans_mat

available inputs: `abstract_field_support` (AbstractFieldSupport), `unit_system_or_result_info` (ResultInfo, string, Int32), `skip_missing_materials` (bool), `Engineering_data_file` (DataSources)

available outputs: `materials_container` (MaterialsContainer), `skipped_materials` (StringField)

**DPF Framework Reference:** [ans_mat_materials_provider operator specification](https://developer.ansys.com/docs/dpf-framework-2027-r1/operator-specifications/geo/ans_mat_materials_provider.md)

**Parameters:**

* **abstract_field_support**
* **unit_system_or_result_info**
* **skip_missing_materials**
* **Engineering_data_file**
* **config**

**Example:**

```python
op = ans_mat_materials_provider()

op = ans_mat_materials_provider(abstract_field_support=my_abstract_field_support,unit_system_or_result_info=my_unit_system_or_result_info,skip_missing_materials=my_skip_missing_materials,Engineering_data_file=my_Engineering_data_file)
```

## Inputs

### abstract_field_support

**Type:** *LinkableInput*

### unit_system_or_result_info

The material properties are converted into this unit system.

**Type:** *LinkableInput*

### skip_missing_materials

If TRUE, the operator adds an empty material to the Materials Container if the material was not found in the ENDG file. Otherwise the operator throws an error. Default is FALSE if the pin is not set. The output skipped_materials (pin 1) can be used to check if materials are missing. Empty materials means that it has no properties, no name etc.

**Type:** *LinkableInput*

### Engineering_data_file

Material file generated by Engineering Data (*.engd or *.xml).

**Type:** *LinkableInput*

## Outputs

### materials_container

A materials container containing all information of available properties (property scripting names), vuuid, material physic names read through ans_mat

**Type:** *LinkableOutput*

### skipped_materials

StringField with the DPF material ID as key and the material UUID as value which are missing in the ENGD file.This output is used in combination with the input skip_missing_materials (pin 2).

**Type:** *LinkableOutput*
