\chapter{How to Use the Examples }
\hypertarget{md_Examples_2Readme}{}\label{md_Examples_2Readme}\index{How to Use the Examples@{How to Use the Examples}}

\begin{DoxyPre}
The Examples folder contains some example files that demonstrate how to use DPF.

The DPF API is written using an Hour Glass Pattern. HGP removes the requirement
of linking DPF libraries into user code.

The included example code compiles on both Windows and Linux. It demonstrates
how to use many of the functions within the DPF API.

There is a simple example, CompleteRST.cpp, for loading a Mechanical RST file.
The code demonstrates how to read the mesh and the results.

The ModelTest.cpp file shows how to read more detailed information from RST
files.

The ResultTest.cpp file shows how to read result-\/related data from RST files.

Each of these files contains several self-\/contained examples. In each example,
information is read from a file and compared to expected values. When compiled and run, the example
code produces output similar to the following:

\begin{DoxyPre}
\begin{DoxyEnumerate}
\item Change to the Examples folder.
\item Load the Example.vcxproj file into Visual Studio.
\item Build the solution. DPF\_Examples.exe is created.
\item Press F5 in Visual Studio to run the code. If an error occurs, the program
   will pause after running and show the results of running the examples.
\end{DoxyEnumerate}\end{DoxyPre}



\begin{DoxyPre}If DPF was installed as part of a normal Ansys Software installation, the
required .dll files are available for running the DPF example code.  The vcxproj
file sets the working folder to the build folder and adds appropriate folders
to the PATH before running the code (location of DataProcessingCore.dll must be 
known by the system PATH).\end{DoxyPre}

\begin{DoxyPre}
\begin{DoxyEnumerate}
\item Change to the Examples folder.
\item Run the command "{}make"{}.  This builds the DPF\_Examples executable.
\item Run the command "{}make run"{}. This runs the DPF\_Examples executable.
\item Run the command "{}Make clean"{}, to clean up.
\end{DoxyEnumerate}\end{DoxyPre}



\begin{DoxyPre}If DPF was installed as part of a normal Ansys Software installation, the
required .so files are available for running the example code. The Makefile sets
the working folder to the build folder and configures LD\_LIBRARY\_PATH and
LD\_PRELOAD before running the code.\end{DoxyPre}



\begin{DoxyPre}Notes regarding LD\_LIBRARY\_PATH and LD\_PRELOAD:
\begin{DoxyEnumerate}
\item LD\_LIBRARY\_PATH must be set to target the location of libDataProcessingCore.so.
\item LD\_PRELOAD must be set to target the libstdc++.so.6, libiomp5.so, libmkl\_core.so and libmkl\_intel\_thread.so locations.
\end{DoxyEnumerate}\end{DoxyPre}



\begin{DoxyPre}Facing issues when running the LD\_PRELOAD related command, edit the Makefile. In the run command, replace the following line:
    export LD\_LIBRARY\_PATH=/dpf/bin/linx64:
    export LD\_PRELOAD=/dpf/bin/linx64/libiomp5.so:/dpf/bin/linx64/libmkl\_core.so:/dpf/bin/linx64/libmkl\_intel\_thread.so:/tp/stdc++/libstdc++.so.6:
These lines must be replaced with the following content:
    export LD\_LIBRARY\_PATH=/aisol/dll/linx64:
    export LD\_PRELOAD=/tp/IntelCompiler/2023.1.0/linx64/lib/intel64/libiomp5.so:/tp/IntelMKL/2024.2.0/linx64/lib/intel64/libmkl\_core.so:/tp/IntelMKL/2024.2.0/linx64/lib/intel64/libmkl\_intel\_thread.so:/tp/stdc++/libstdc++.so.6:\end{DoxyPre}

\begin{DoxyPre}The example code looks for data files relative to the build folder. You must be
in the build folder before running the example executable.\end{DoxyPre}



\begin{DoxyPre}The data files are in the "{}testfiles"{} folder at the DPF root.\end{DoxyPre}



\begin{DoxyPre}The example code requires many common files from Ansys to run, so DPF must be
installed with Ansys software.
