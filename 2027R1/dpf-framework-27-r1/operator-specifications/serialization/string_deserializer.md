# serialization:string deserializer

**Version: 0.0.0**

## Description

Takes a string generated by the serializer and deserializes it into DPF's entities.

## Inputs

| Pin number | Name | Expected type(s) |
|-------|-------|------------------|
| <strong>-1</strong> <span style="background-color:#d93025; color:white; padding:2px 6px; border-radius:3px; font-size:0.75em;">Required</span>|  [stream_type](#input_-1) |[`int32`](../../core-concepts/dpf-types.md#standard-types) |
| <strong>0</strong> <span style="background-color:#d93025; color:white; padding:2px 6px; border-radius:3px; font-size:0.75em;">Required</span>|  [serialized_string](#input_0) |[`string`](../../core-concepts/dpf-types.md#standard-types) |


<a id="input_-1"></a>
### stream_type (Pin -1)

- **Required:** Yes
- **Expected type(s):** [`int32`](../../core-concepts/dpf-types.md#standard-types)

0 for string (default), and 1 for binary

<a id="input_0"></a>
### serialized_string (Pin 0)

- **Required:** Yes
- **Expected type(s):** [`string`](../../core-concepts/dpf-types.md#standard-types)




## Outputs

| Pin number |  Name | Expected type(s) |
|-------|------|------------------|
|  **1**| [any_output](#output_1) |[`any`](../../core-concepts/dpf-types.md#any) |


<a id="output_1"></a>
### any_output (Pin 1)

- **Expected type(s):** [`any`](../../core-concepts/dpf-types.md#any)

number and types of outputs corresponding of the inputs used in the serialization


## Configurations


### mutex

- **Expected type(s):** [`bool`](../../core-concepts/dpf-types.md#standard-types)
- **Default value:** false

If this option is set to true, the shared memory is prevented from being simultaneously accessed by multiple threads.

## Examples

<details>
<summary>C++</summary>

```cpp
#include "dpf_api.h"

ansys::dpf::Operator op("string_deserializer"); // operator instantiation
op.connect(-1, my_stream_type);
op.connect(0, my_serialized_string);

```
